1 פתח את Clerk Dashboard

היכנס לדשבורד של Clerk ובחר את אפליקציית Sipurai.

🔗 פתח Clerk Dashboard
💡 טיפ: אם יש לך כמה אפליקציות, ודא שאתה בוחר את האפליקציה הנכונה — זו שמקושרת ל-sipurai.ai.

2 עבור למצב Production

בדשבורד של Clerk, יש toggle בין Development ל-Production בפינה השמאלית העליונה (או בתפריט הצדדי).

  1. לחץ על "Production" כדי להפעיל את ה-Production Instance
  2. Clerk ייצור אוטומטית מפתחות חדשים עם prefix של pk_live_ ו-sk_live_
  3. ייתכן שתתבקש לאשר DNS או domain — עקוב אחרי ההוראות
🖼️ חפש את ה-Toggle שכתוב עליו "Development" ולחץ כדי לעבור ל-"Production"
⚠️ שים לב: לאחר המעבר ל-Production, מפתחות ה-Test ימשיכו לעבוד רק בסביבת Development. באתר החי חייבים להשתמש במפתחות pk_live_.

3 העתק את המפתח החדש

לאחר שעברת למצב Production:

  1. לך ל-API Keys בתפריט הצדדי
  2. העתק את ה-Publishable Key — הוא יתחיל ב-pk_live_
  3. שמור גם את ה-Secret Key (sk_live_) למקרה שתצטרך בעתיד
# המפתח הישן (לא עובד בפרודקשן):
VITE_CLERK_PUBLISHABLE_KEY=pk_test_xxxxxxxxxxxxx

# המפתח החדש (יעבוד בפרודקשן!):
VITE_CLERK_PUBLISHABLE_KEY=pk_live_xxxxxxxxxxxxx
⚠️ חשוב: אל תשתף את ה-Secret Key (sk_live_) בשום מקום ציבורי. ה-Publishable Key בטוח לשימוש בקוד צד-לקוח.

4 עדכן ב-Vercel

עדכן את משתנה הסביבה ב-Vercel כדי שהאתר החי ישתמש במפתח החדש.

🔗 פתח Vercel Dashboard
  1. בחר את פרויקט sipurai
  2. לך ל-Settings → Environment Variables
  3. מצא את VITE_CLERK_PUBLISHABLE_KEY
  4. לחץ על Edit (סמל העיפרון)
  5. עדכן את הערך למפתח החדש pk_live_...
  6. ודא שה-Variable מוגדר ל-Production (ואם תרצה, גם Preview ו-Development)
  7. לחץ Save
💡 טיפ: אם יש לך גם CLERK_SECRET_KEY ב-Vercel (לצד שרת), עדכן גם אותו ל-sk_live_...

5 עדכן .env מקומי

עדכן את קובץ .env בפרויקט המקומי כדי שגם סביבת הפיתוח תעבוד.

# .env (בתיקיית הפרויקט)

VITE_CLERK_PUBLISHABLE_KEY=pk_live_YOUR_NEW_KEY_HERE
💡 טיפ: ניתן לשמור גם את מפתח ה-Test בקובץ .env.development נפרד אם רוצים סביבת Development עם Clerk Test ו-Production עם Live.

6 הגדר URLs בדשבורד Clerk

ב-Clerk Dashboard, במצב Production, הגדר את ה-URLs הבאים:

לך ל-Paths (או URLs) בתפריט הצדדי.

Sign-in URL/sign-in
Sign-up URL/sign-up

After sign-up URL/

בנוסף, ב-Domains או Allowed Origins, הוסף:

https://sipurai.ai

⚠️ חשוב: Clerk ב-Production דורש domain מאומת. אם תתבקש להגדיר DNS records (CNAME), עקוב אחרי ההוראות בדשבורד.

7 הפעל שיטות התחברות

ב-Clerk Dashboard, לך ל-User & Authentication → Email, Phone, Username ו-Social Connections.

ודא שהשיטות הבאות מופעלות:

שיטהסטטוסהערות
Email חובה שיטת ברירת המחדל
Google OAuth מומלץ מאוד רוב ההורים משתמשים ב-Google
Apple אופציונלי למשתמשי iPhone/iPad
💡 Google OAuth: ב-Production, תצטרך Google OAuth Client ID מ-Google Cloud Console. צור OAuth 2.0 Client ID עם ה-redirect URI שמופיע בדשבורד של Clerk.

8 Redeploy ובדוק

אחרי שעדכנת את כל הערכים:

  1. ב-Vercel, לחץ על Deployments → בחר את ה-Deployment האחרון → Redeploy
  2. חכה שה-Deployment יסתיים (1-2 דקות)
  3. גש ל-https://www.sipurai.ai/sign-in
  4. ודא שדף ההתחברות נטען ללא שגיאות
  5. נסה להתחבר עם Email ו/או Google
✅ בדוק את דף ההתחברות
✅ איך תדע שזה עובד: דף ההתחברות יופיע עם ממשק Clerk, בלי שגיאות בקונסול, ותוכל להתחבר בהצלחה.
